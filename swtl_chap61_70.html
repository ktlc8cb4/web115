Chapter 61 Concepts: The DOM: Junk Artifacts and nodeType.

Reminding that Nodes are the elements in the the DOM. However, the browsers will read white spaces such as tabs or carriage returns as Nodes, but this is incorrect. This chapter shows several methods to solve this problem.

Syntax

Method 1: Suggested by Mozzila Developer Network, we should carry the carriage to the next line, eliminating the problem of carriage returns. Take a look at the syntax below of an HTML file.

Notice the closing carriage characters are on the next line. This eliminates the whitespace created by the carriage returns and prevents the browers to see it as a Node.

<html
><head
><title
>Title of Page
</title
></head
><body
><div
><p
>Some paragraph content
</p
></div
></body
></html>

Method 2: Use a minifier to clean up all the spaces in the HTML file. Take a look at the an HTML file after minified.

Notice that there are no returns. Every element is chained to the previous element wihth no space. There are minifier programs we can use to achieve this; however, we should back up our files before using the minifier to preserve the original file because the minified version is obviously very hard to read through to make changes.

<html><head>some content</head><body><div>
<p>some text</p></div></body></html>

Method 3: Using nodeType.

nodeType is used for Javascript to identify what kind of node an element is. It returns a specific number depending on the type of node it finds. A <div> and <p> will return 1, a text node will return 3. The syntax below shows how we can use nodeType to target a specific element. 

// The HTML block

<div id="id">
  <p> some text </p>
  <p> some text </p>
  <p> some text </p>
</div>

/* Script to target the second paragraph
   and change its content */

var aVar = 
  document.getElementById("id");
var counter = 0;
for(var i=0;i<aVar.childNodes.length;i++){
 if(aVar.childNodes[i].nodeType===1){
  counter++;
 }
 if(counter===2){
 aVar.childNodes[i].innerHTML = "content";
 break;
 }
}

Example

We will use the syntax above to give this example to change the seccond paragraph in a div

Chapter 62 Concepts: The Dom: More Ways to Target Elements.

This chapter shows us a few more ways to target elements using several methods: lastChild, firstChild, parentNode, nextSibling, previousSibling.

Syntax

We can use all these methods using the syntax shown belown.

var aVar = parentNode.firstChild;
var aVar = parentNode.lastChild;
var aVar = element.nextSibling;
var aVar = element.previousSibling;
var aVar = childNode.parentNode;

Notes to remember:

  • lastChild is used to target the last child node of that parent element.
  • firstChild is used to target the first child node of that parent element.
  • nextSibling is used to target the next child from the named child node of the same parent element.
  • previousSibling is used to target the previous child node from the named child node of the same parent.
  • parentNode is used to get the parent node of the named child node.

Example

The example below targets the last paragraph of the div element and change its class.

// HTML block

<div id="deerMission">
  <p>Deer are gentle.</p>
  <p>Deer are cute.</p>
  <p>Deer are vegetarians.</p>
</div>

// Script

var aVar = 
 document.getElementById("deerMission");
aVar.lastChild.className += " redDeer";

Demonstration

Chapter 63 Concepts: The Dom: Getting a Target's Name.

This chapter shows us how to get the target element's name by using nodeName.

Syntax

Note that the nodeName methods returns the node name of an element in all uppercase letters. It returns the name of a text node in lowercase letters.

We can also get the value of the text node by using nodeValue

// To get node name

var aVar = targetElement.nodeName;

// to get a text node value

var aVar = textNode.nodeValue;

Note that nodeValue is not the same as innerHTML. innerHTML gets all the inner content of an element. nodeValue only shows the value of that one text node in that element.

Example

The example below will return the name of the first paragraph, which should be P.

  <p>Deer are vegetarians.</p>
</div>

// Script

 document.getElementById("deerMission");

Chapter 65 Concepts: The Dom: Attributes.

Besides the children nodes an element can have, the element also has attributes. This chapter shows us how to work with attributes by using these methods: hasAttribute, getAttribute, setAttribute.

Syntax

Use hasAttribute to find out if an element has a certain attribute. If it does, it will return true.

Use getAttribute to read the value of an attribute.

Use setAttribute to set the values for an attribute.

// The hasAttribute method

var aVar = element.hasAttribute("name");

// The getAttribute method

var aVar = element.getAttribute("name");

// The setAttribute method

var aVar = 
 element.setAttribute("name","value");

Notes to remember:

  • "name" inside the parentheses is the name of the attribute of interest.
  • For setAttribute, "name" is the name of the attribute we want to set. "value" is the value we want to give to that attribute.

Example

In this example, we will set an attribute of class to our first paragraph.

// The HTML block

<div id="deerMission">
  <p>Deer are gentle.</p>
  <p>Deer are cute.</p>
  <p>Deer are vegetarians.</p>
</div>

// Script

var aVar =
 document.getElementById("deerMission");
var firstP = aVar.firstChild;
firstP.setAttribute("class","redDear");

Demonstration

We will use the example above to set a class for the first paragraph of the div.

Chapter 66 Concepts: The Dom: Attribute Names and Values.

We can compile a list or an array of attributes of an elements. After the list is made, we can count the number of attributes that element has. We can also read the name of that attribute and its value.

Syntax

Note that the returned name of the attribute is in lowercase.

// To make a list of attributes

var aVar = 
 element.attributes;

// To get the name of an attribute

var aVar = element.attribute.nodeName;

// To get the value of an attribute

var aVar = element.attribute.nodeValue;
