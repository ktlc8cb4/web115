Chapter 71 Concept: Objects: Methods.

An object property can have a value that is a function. That function is an object method.

Syntax

The syntax shows a function created within an object which becomes the object method.

var object = {
 objectProperty1: value,
 objectMethod: function(){
   function code;}
};

Notes to remember

  • The function written withing an object uses the same syntax as other functions.
  • If there is another property following the method, we need to add a comma at the end of the method just like a normal property.
  • If there the method uses one of the object's property, then use "this" keyword as shown below to give an example of the method using the same object's property.
var object = {
 objectProperty1: value,
 objectMethod: function(){
   return this.objectProperty1;}
};

Example

This example will create a simple Deer object that records Deer profile. It will have a method that returns a discount amount in dollars if our Deer is a senior with the age above 65. The discount will be 5 dollars plus the percent discount multiplies the age of our Deer. The users will have to put in the discount percent.

 name: "John",
 age: 70,
 favDish: "Avocado Salad";
 deerDiscount: function(percentDisc) {
  if (this.age > 65) { 
   return(5 + ((percentDiscount*this.age)/
                100));
  }
 }
};

// Calling the method

var discAmount = deer.deerDiscount(80);

Chapter 72 Concepts: Objects: Constructors.

This chapter shows us how to write a function that will serve as a constructor to mass produce objects of the same kind.

Syntax 

// Creating the constructor

function Object(parameter1,
                         parameter2) {
 this.property1 = parameter1;
 this.property2 = parameter2;
}

// Creating an object using Constructor

var object1 = 
 new Object(parameter1, parameter2);

Notes to remember:

  • The name of this constructor function starts with a capital letter.
  • Each of the parameter is assigned to a property of the object that will be created.
  • It is common to make the property name to be the same as the parameter name. In the above syntax, we did not use the same names.

Example and Demonstration

With this chapter, we combine the example and demonstration into one section because there is no clickable demonstration to show.

 this.name = name;
 this.age = age;
 this.favDish = favDish;
}

// Creating an new Deer profile

var deer1 = new Deer("John", 23, "Salad");

Chapter 73 Concepts: Objects: Constructors and Methods.

This chapter shows us how to create a constructor with a function inside it. When we create a object using this constructor, the function becomes the object method.

Syntax 

// Creating the constructor

function Object(parameter1,
                         parameter2) {
 this.property1 = parameter1;
 this.property2 = parameter2;
 this.method = function(mParameter){
  code to be executed;
 };
}

// Creating an object using Constructor

var object1 = 
 new Object(parameter1, parameter2);

// Calling the method of that object

object1.method(mParameter);

Notes to remember:

  • Creating the method within a construction is the same as creating the method within an objecct. The only difference is the "this" keyword.
  • With the inclusion of this method inside the constructor, every new object created will have this method.
  • The parameter inside the method is not passed when the constructor is called to create a new object. This parameter will only need to be passed when calling that method.

Example

The example below will create a constructor Deer with a method. Then, it will create a object and call the method within that object which is a method that will return the discount if our Deer is a senior over 65 years of age.

// Creating a constructor

function Deer(name, age, favDish) {
 this.name = name;
 this.age = age;
 this.favDish = favDish;
 this. deerDiscount = function() {
  if (this.age > 65) { 
   return Math.round(5 + ((5*this.age)/
                100));
  }
 };
}

// Creating an new Deer profile

var deer1 = new Deer("John", 23, "Salad");

// Calling the method

var discAmount = deer1.deerDiscount()

Demonstration

We will use the example above to create a new Deer profile object, and return a discount amount according to the Deer's age.

Chapter 74 Concepts: Objects: prototypes.

When we write a method inside a constructor, it will write this method for every new object created by that constructor. This is too many duplicates. This chapter shows us to write a prototype for the object constructor which will be shared by as many instances of that constructors as been created.

Syntax 

Object.prototype.prototypeMethod = 
 function() {
  codes within the constructor;
}

// Creating a prototype property

Object.prototype.property = value;

Notes to remember:

  • We don't include the method when we create the constructor. We add the above syntax after the constructor has been created.
  • Prototypes can be applied to both methods and properties as seen above.

Demonstration and Example.

Chapter 75 Concepts: Objects: Checking for Properties and Methods.

We can check to see if an object has a certain property or method.

Syntax 

// Check for property

var aVar = "property" in object;

// Check for method

var Avar = "method" in object;

/* Check to see all the properties 
   of an object

var listOfProperties = [];
for (var prop in object) {
 listOfProperties.push(prop);
}

// Using hasOwnProperty

var aVar = 
 hasOwnProperty("property");

// Using hasOwnProperty to get a list

var listOfProperties = [];
for(var prop in object) {
 if(object.hasOwnProperty(prop)){
  listOfProperties.push(prop);
 }
}

Note to remembers:

  • When checking to see if an object has the property or method, the return value is true or false.
  • The hasOwnProperty technique only shows the properties or methods that were declared when the object was created, or the properties and methods that came with the original constructor. The prototype or inherited properties/methods.
  • The var prop inside the parentheses doesn not have to be "prop", it can be any other names.

Example

Chapter 77 Concepts: Browser Control: Getting and Setting the URL Another Way.

This chapter shows us how set or get the URL with replace(), assign(), and reload().

Syntax 

// Using replace()

window.location.replace("URL");

// Using assign()

window.location.assign("URL");

// To reload the current page 

window.location.reload(true);
window.location.reload();
window.location.reload(false);
window.location.href = 
 window.location.href;

Notes to remember;

  • The assign() allows the users to backspace to the previous page.
  • The replace() does not allow the users to backspace to the previous page. If the user uses backspace, it will direct the users to the page previous to the previous page.
  • If the argument of the reload() is true, the browser reloads the page from the server.
  • If the argument of the reload() is false, the browser reloads the page from the cache if the page has been cached.

Example

Chapter 78 Concepts: Brower Control: Forward and Reverse.

This chapter shows us how to direct the browser to go backward one page in history or forward one page in history when the users press backspace or click on the back button. We can also get the URL of the page where the users has clicked on a link to get to another page.

If there is no page forward or backward in history, the browser does nothing.

You can use a number to direct the browser to go back or forward by that number of pages. A negative number will direct the browser to go backward. A positive number directs the browser to go forward.

Syntax 

// To go backward in history

history.back();

// To go forward in history

history.forward();	

// To go backward to 3 pages in history

history.go(-3);

// To go forward to 3 pages in history

history.go(3);

/* To get the URL of the page where a 
   link was clicked.

var aVar = document.referrer;

Notes to remember:

  • When using the go(number) method, there is no reliable way to know how many pages forward or backward are in the history.
  • The referrer method only works if a link was clicked. if the users get to the current page through a bookmark or by entering the URL, Javascript will not return the URL of the page where users came from. It will return an empty string.

Example

This example shows how to go back to the previous page.

history.back();

Chapter 79 Concepts: Browser Control: Filling the Window with Content.

This chapter shows us how to create a pop up window and fill it with content.

Syntax 

// Creating a popup window

var popUp = window.open();

// Creating content

var content = "content";

// Filling the popup with content

popUp.document.write(content);
popUp.location.assign("URL to content");
popUp.location.href = "URL";

// To close the popUp window

popUp.close();

Notes to remember:

  • Different browsers open the popup window at different locations or a new window. We cannot control this.
  • When writing the content for the variable content, anything inside the content that requires quotation marks need to be in single quotes.
  • The document.write method is used to fill an empty window.

Example

This example creates a popUp windown with a greeting.

 window.open(`Hello, 
             Welcome to The Vegan Deer!`);

Chapter 80 Concepts: Browser Control: Controlling the Window's Size and Location.

This chapter shows us how to control the size of the popup window.

Syntax 

var popUp =
 window.open("URL","win1",
             "width=400,
             height=400,
             left=200,
             top=200"
             );

Notes to remember:

  • win1 is a name parameter. It cannot have any spaces in it.
  • The width, height, left, top are parameters to control the size and the location of the popup window. It cannot have any spaces in it. The different parameters in this sextion do not have to follow any specific order.
  • The number 400 and 200 in the syntax are just examples. It can be any number of our choice. These numbers are in pixels.
  • In order to specify these parameters, you have to specify the URL and a name first.
  • We can specify the window size without the window position, but we cannot specify the window position without the size. It will be ignored.
  • The whole popup window specifications can be saved to a variable and be used in the open() parentheses.
