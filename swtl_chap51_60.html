Chapter 51 Concepts: Reading and Setting Paragraph Text.

We can use innerHTML to read or to set text for an HTML element.

Syntax

We assign the content we read from an HTML element to a variable to capture the information. We also need to give the targeted element an ID.

var someVar =
  document.getElementById("ID").innerHTML;

To set the content for an HTML element, we also need to give that element an ID. 

document.getElementById("ID").innerHTML =
  content;

Example

Setting text for an HTML element. Assuming the HTML element has an ID of "myDeer".

document.getElementById("myDeer")
  .innerHTML = "Hello, my Deer!";

Getting text from an element with the ID of "myDeer" and assign that content to a variable named deerGreeting.

var deerGreeting = 
  document.getElementById("myDeer")
    .innerHTML;

Chapter 52 Concepts: Manipulating Images and Text.

We can use .className to set a class for an HTML element.

Syntax

To set a class for an HTML element, it is required that there is a class defined in CSS. Again, since we are using the document.getElementById method, it is required that the HTML has a set ID.

Using the method shown in the syntax below will erase all classes that are currently set for the HTML element.

document.getElementById("ID").className =
  "class";

Using the method shown in the syntax below will preserve the currently set classes for the HTML element. Notice the difference is the extra + sign in front of the equal sign and the space before the " class" declaration.

document.getElementById("ID").className +=
  " class";

Example

In the example below, we will make our Deer turn red when we click on them.

/* CSS: defining a class */
.redDeer { background-color: red;}

/* HTML markup - using an h2 element */
<h2 id="reddeer" onclick="redDeer()">
  The Vegan Deer
</h2>

/* Scripts */
function redDeer() {
  document.getElementById("reddeer")
    .classname = "redDeer";
}

Demonstration

Chapter 53 Concepts: Swapping Images.

We can use .src to swap images when an event is triggered on an img element.

Syntax

In the syntax below, the image inside the quotation marks is the image used to replace the existing image.

document.getElementById("ID"). src =
  "image";

Example

Switching an image of a deer to a deer eating grass.

/* HTML markup */
<img src="deer.jpg" id="before"
  onclick="changePic();">

/* Scripts */
function changePic() {
  document.getElementById("before")
    .src="deereatinggrass.jpg";
}

Demonstration

We will use the example above to change an image from a deer to a deer eating grass.

Chapter 55 Concepts: Setting Styles.

we can use .style to set or read the styles of an HTML element.

Syntax

The syntax below is to set or to read the styles for the HTML element.

// To set styles
document.getElementById("ID").style
  .property =
  "value";

// To get styles
var someVar = 
  document.getElementById("ID").style
    .property;

Points to remember:

  • You can style many HTML properties using this technique.
  • It's best to look up the correct keywords for the properties.
  • Reading styles can be tricky since some browsers do not support it, or the code only specifies for the document to read the inline styles only.

Example

In this example, we use .style.property to make our Vegan Deer heading grow bigger.

// HTML markup
<h2 id="bigger" onclick="makeBigger()">
  The Vegan Deer
</h2>

// Scripts
function makeBigger() {
  document.getElementById("bigger")
    .style.fontSize = "3em";
}

Chapter 56 Concepts: Target All Elements by Tag Name.

We can target HTML elements by using their tag names.

Syntax 

document.getElemensByTagName("tagname");

Points to remember:

  • The elements targeted by using getElementsByTagName are stored like an array.
  • We can access the elements in this array by indexing.
  • We can get or set information from or to the elements by accessing them using the array indexing technique.

Example

This example shows how we get all paragraph elements and change the background color for the third paragraph

/* HTML markup */
<!DOCTYPE html>
<html>
<head></head>
<body>
 <p>The deer is a gentle creature.</p>
 <p>Deer are cute</p>
 <p onclick="changeParaColor">
		Deer are vegan!
	</p>
</body>
</html>

/* Scripts */
function changeParaColor() {
 var deerPara =
  document.getElementsByTagName("p");
 deerPara[2].style.backgroundColor = 
  "red";
}

Chapter 57 Concepts: Target Some Elements By Tag Name.

When we use document.getElementsByTagName, we select all the elements with that tag name in the whole document from the beginning to the end. This chapter shows us how to only get a elements by tag name in a specific section of the document.

Syntax

To get elements by tag name in one specific section, we use a <div> with a specific ID. We first getElementById for that specific div. Then we can get all elements by tag name in that div.

We use the <p> elements in this syntax; however, this technique applies to all HTML elements.

Also, we store the value from document.getElementsByTagName to a variable. We don't always have to do that depending on how we use the document.getElementsByTagName values.

/* HTML markup */
<!DOCTYPE html>
<html>
<head></head>
<body>
 <div id="id">
  <p>Content</p>
  <p>Content</p>
  <p>Content</p>
 </div>
</body>
</html>

/* Scripts */
var someVar = 
  document.getElementsById("id");
var someOtherVar =
  someVar.getElementsByTagName("p");

Example

Because this section is similar to the last section, we will use the same example to highlight the third paragraph; however, it differs in where we add the <div> element, so we can target its ID. 

/* HTML markup */
<!DOCTYPE html>
<html>
<head></head>
<body>
 <div id="deerDiv">
  <p>The deer is a gentle creature.</p>
  <p>Deer are cute</p>
  <p>Deer are vegan!</p>
 </div>
</body>
</html>

/* Scripts */
function changeParaColor() {
 var deerSection = 
  document.getElementById("deerDiv");
 var deerPara =
  deerSection.getElementsByTagName("p");
 deerPara[2].style.backgroundColor = 
  "red";
}

Chapter 58 Concepts: The DOM.

Points to remember:

  • DOM stands for Document Object Model.
  • The DOM is an organizational chart created automatically by the browser.
  • Every element in an HTML Document, which is a web page, has a spot on this organizational chart.
  • We can compare the DOM to a company organizational chart that includes everyone from the CEO down to all employees. Like a company who allows an outsider to contact any employee if the outsider knows that employer's email address or phone number or some form of employee identification, the DOM gives each of its elements an "address" to allow the outsider, Javascript in this case, to access every element inside that DOM as long as Javascript names the correct "address".
  • Each element inside a DOM is called a Node.

Example

There is no code examples for this chapter. Below is an HTML document. This example just shows what the Nodes of this document are.

/* HTML markup */
<!DOCTYPE html>
<html>
<head></head>
<body>
 <div id="deerDiv">
  <p>The deer is a gentle creature.</p>
  <p>Deer are cute</p>
  <p>Deer are vegan!</p>
 </div>
</body>
</html>

Each element within the pair of < > tags is a Node in this HTML document.

In the above example, the nodes are the <html>, <head>, <body>, <div>, and all three <p> elements.

Chapter 59 Concepts: The DOM: Parents and Children.

Points to remember:

  • To review: Each element in the DOM is called a Node.
  • Each node can be considered a parent of some other nodes, and each node can also be a child of another node.
  • A node is a child of the node the encloses it.
  • Nodes like <em>, <strong>, <span> are also nodes that are the parents of the text within their enclosure.

Example

We will use the same HTML markup for the last chapter's example to show the parents and the children in the document.

/* HTML markup */
<!DOCTYPE html>
<html>
<head></head>
<body>
 <div id="deerDiv">
  <p>The deer is a gentle creature.</p>
  <p>Deer are cute</p>
  <p>Deer are vegan!</p>
 </div>
</body>
</html>

Below is the list of the parent and child relationship in the example

  • The top most level of the DOM is the <html> element.
  • The <head> is the first child of <html>, the <body> is the second child of <html>. In reverse, <html> is the parent of both <head> and <body>.
  • The <div> is the first child of <body>. In reverse, <body> is the parent of <div>.
  • The three <p> are first, second, and third child of <div> in the order they appear in the document. In reverse, <div> is the parent of all three <p> nodes.

Chapter 60 Concepts: The DOM: Finding Children.

We can use .childNode[] to target specific HTML element in the DOM per the parent-children relationship.

Syntax 

document.childNodes[index]
        .childNodes[index]
        .childNodes[index];

Points to remember:

  • The index is the number of its order in the enclosure of its parent.
  • In the syntax above, each childNode is the children of the previous childNode. The syntax only shows 3 childNodes, but we can chain as many childNode as necessary to get to our targeted element.
  • Children are stored in an array. So we place the index number like that of an array inside the square brackets to access the childNode we want.

Example

We will use the simple HTML document in the previous example to access the third paragraph within the body using the .childNode[] technique.

/* HTML markup */
<!DOCTYPE html>
<html>
<head></head>
<body>
  <p>The deer is a gentle creature.</p>
  <p>Deer are cute</p>
  <p>Deer are vegan!</p>
</body>
</html>

/* Scripts */

var deerPara =
 document.childNodes[0].childNode[1]
         .childNodes[2];
