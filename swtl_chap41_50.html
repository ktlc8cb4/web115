Chapter 41 Concepts: While Loops.

While Loop is similar to For Loop. They have different syntax.

Syntax 

var counter = 0;
while(condition) {
  Code to be executed;
  counter++;
}

A few points to remember:

  • var counter is initialized with a value before the loop.
  • The counter is updated within the code.

Example

This example will welcome our new Deer 5 times.

var i = 0;
var text = "";
while(i < 5) {
  text += "Welcome to The Vegan
               Deer!";  
  i++;
}
console.log(text);

Chapter 42 Concepts: Do...While Loops.

Do...While Loop is similar to while loop. The only difference is Do...While Loop will execute the code at least once even if the condition doesn't satisfy.

Syntax

Note that the while(condition) goes behind the do loop code.

var counter = 0;
do { 
  code to be executed;
  i++;
} while(condition);

Example

The condition for the code below will never be true; however, the code will print out our favorite ingredient at least once.

var i = 6;
var text = "";
do {
  text += "cheese";
  i++;
} while(i<5);
console.log(text);

Demonstration

We will try the do...while loop in our example to show that the favorite ingredient will be printed out at least one time.

Chapter 43 Concepts: Placing Scripts.

Scripts can be placed any where in the html file. Pieces of scripts can be scattered everywhere in an html file if that is what the programmer wants to do. A separate script file can be linked to an html file, which is the preferred method. Whether linking a separate file or embedding scripts within the html files, the preferred placement is at the bottom of the body so to allow CSS and HTML elements time to load before the scripts are loaded.

Syntax

Placing a script inside an html file, use a pair of scipt tags.

<script> codes </script>

Linking a separate script file. 

<script src="scriptfile.js"></script>

Chapter 44 Concepts: Commenting.

Comments are parts of code that are written for humans not machines. The code ignores them. While not being a part of the code, they are very important because they are remarks about the codes. In a team collaboration and debuggin, they are extremely important because programmers can read them to figure what the codes are written for.

Syntax

Use two forward slashes for one line comments.

// One line comment. Just one line.

Use forwatd slash and star to enclose multi-line comments.

/* One line of comments. 
   another line of comments.
   and another line of comments.
*/

Chapter 45 Concept: Event: Link.

When users interact with a website, it is called an event. The Javascript codes that respond to those events are called event handlers. This chapter focuses on the onClick event handler for the link element.

Syntax

Using links to establish a place for user interactivity. The link below does not take a user to a link perse, it just refers to the event handler to respond accordingly.

<a href="#"
  onclick = "code to be execcuted;">
  link texts or images
</a>

Because it being a link, when the user clicks on it, it will refresh the page and take the users all the way to the beginning of the page. The users then have to scroll down to where they were. The syntax below will help with that problem.

<a href="JavaScript:void(0)"
  onclick = "code to be execcuted;">
  link texts or images
</a>

Example 

<a href="JavaScript:void(0)" 
  onclick="alert('Welcome, Deer');">
A Deer's Greetting
</a>

Notice that we set the code to be executed inside a quotation marks for the value of onclick. This is called an inline event handlers where the code is embedded as the value for onclick. There is a more professional way which the author will show us in a later chapter.

Chapter 46 Concept: Events: Buttons.

For interactivity and event handling, using buttons for users to click gives it a more professional look.

Syntax

Use input tag to create a button.

<input type="button" 
  value="value" 
  onclick="code to be executed">

Use an image in place of a button. 

<img src="imagefile" 
  onclick="code to be executed">

Example

Using input to create a button that handles an event by displayng an alert message.

<input type="button"
  value="click me"
  onclick="alert('Hello there!');">

Chapter 47 Concepts: Events: Mouse.

Use onMouseover and onMouseout to respond to the interaction when users hover their mouse over an element and then move their mouse away from that element.

Syntax

onMouseover and onMouseout usually work together. When the users hover above an html element, onMouseover will execute its code. Then when the users move away from that html element, onMouseout will execute its code.

/* Showing syntax for both methods
   when paired together to handle 
   an event. */

<html element 
  onMouseover="code to be executed"
  onMouseout="code to execute">
  element text if any
</html element>

Points to remember:

  • The code that is executed can be written directly inside the quotaion mards following the event handler.
  • The code that is executed can also be a calling of a function on a separate file.

Example

An example to display an alert message with an h2 element.

<h2 onMouseover=
  "alert('Let's go to The Vegan Deer');">
  Where are we going for dinner?
</h2>

An example of displaying different images using the pair of onMouseover and onMouseout.

<img src="deer.jpg" 
  onMouseover="src='bambi.jpg'"
  onMouseout="src='deer.jpg'">

The above example will revert display bambi.jpg when users hover the mouse over the image then change the display from bambi.jpg back to deer.jpg when users move the mouse away.

Chapter 48 Concepts: Events: Fields.

Using onFocus and onBlur as event handlers for form fields.

Syntax

Similar to onMouseover and onMouseout, onFocus and onBlur are used together to handle events when users click on a form field and move away from the field respectively.

<input type="formtype"  
  onFocus="code to be executed"
  onBlur="code to be executed">

Example

The code to be executed can either be written inline or a function being called from a separate script. The second approach is more professional.

In this example, we will use inline code. 

<input type="text" 
  onFocus="this.style.backgroundColor=
    '#ffe4b5';"
  onBlur="this.style.backgroundColor=
    '#c9ffe2';">

Chapter 49 Concepts: Reading Field Values.

How to read field values which are the information users enter into a form field.

Syntax

One event handler for form fields is onSubmit. The syntax is below. Notice the onSubmit event handler is on for <form> tag, not on the submit button.

<form onSubmit="code to be executed;>
  <input type"formtype">
  <input type="submit" value="submit">
</form>

Sometimes, the code to be executed from the event handler needs to read the values the users' input. The syntax to read those values is shown below. 

document.getElementById("id").value;

We need to make sure we give the HTML element an ID, in this case it is usually an input element. The document.getElementById("id") is a command that tells the document to choose the element with the ID we specify inside the parentheses.

Example

Using the event handlers and print out the value obtained from users' input. In this example, we will call a function from a separate script.

// HTML code

<form onSubmit="lunch();">
<label for="lunch">
  What's for lunch
</label>
<input type="text" id="lunch" value="">
<input type="submit" value="submit">
</form>

//script function

function lunch() {
  var lunch = 
   document.getElementById("lunch").value;
  console.log(lunch);
}

Chapter 50 Concepts: Setting Field Values.

You can also set the form field values. This is normally used to automatic fill in values for form fields based on other values the users have input. For example, if the users filled in the form with a number of their credit cards, the form can automatically fill in if the credit card is a Master, Visa or others based on the card number the users has put in.

Syntax

We use the same code line as when we get values from the fields. The only difference is we also set it equal to some value we choose. 

document.getElementById("id").value = 
  value;

Reminding that the ID is important. It correlates the ID that belongs to the HTML element. So we must remember to give the targeted HTML element the ID we want to change the value for.

Example

Let's say our Deer has made an order by using a form. If our Deer orders a Veggie burger, the free side is sweet potato fries. If they order anything else, the free side is potato salad. We will automatically fill that in for our Deer.

// HTML code.

<form onSubmit="lunch();">
  <label for="order">
    What's for lunch, Deer?
  </label><br>
  <input type="text" 
         id="order"
         value="Cheese Sandwich"><br>
  <label for="side">
     Your free side is:
  </label><br>
  <input type="text"
         id="side"
         value="">
  <input type="submit" value="Submit">
</form>

// Script 

function lunch() {
  var order = 
   document.getElementById("order").value;
   order = order.toLowerCase();
  if (order === "veggie burger"){
    side = "Sweet potato fries";
  } else {
    side = "Potato salad";
  }
  console.log(order + " " + side);
}
