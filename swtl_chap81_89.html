Chapter 81 Concepts: Browser Control: Testing for Popup Blockers.

This chapter shows us how to test for popup blockers from the browser of the users.

The test will pop up a window so fast the users will not notice it. If the test passes, the window handle will have a value. If it doesn't pass, the handle will have a null value, after which, we can ask the users to allow popups.

Syntax

The syntax belows shows us how to test for popu and ask the users to allow popup if the test fails. 

// Function for popup test

function checkPopUp() {
 var test = window.open("",
                        "",
                        "size if used");
 if(test===null||typeof(test===
                        "undefined"){
  alert("Ask users to disable popup");
 }

// Closing the test popup window

test.close(); 
}

// Running the test in HTML

<body onLoad="checkPopUp();">

Notes to remember

  • All browers returns null if popup test fails except for Internet Explorer. It returns "undefinned" instead of "null". So to cover all bases, we add the part - ||typeof(test==="undefined") to make sure it also works for Internet Explorer.
  • The syntax above also shows how to run the test which is normally run when the page loads.

Example

This example will use the similar code in the syntax to test for popup blockers.

// Function for popup test

function checkPopUp() {
 var test = window.open("",
                        "",
                        "width=100,
                         height=100");
 if(test===null||typeof(test===
                        "undefined"){
  alert("Ask users to disable popup");
 }
test.close();
} 

// Running the test in HTML

<body onLoad="checkPopUp();">

Chapter 82 Concepts: Form Validation: Text Fields.

This chapter shows us how to validate the user input when filling out forms. It focuses on checkign for NAMES such as last names or first names entered by users.

Syntax

The syntax 

// HTML Form Block

<form onSubmit="return checkFunction();">
 <label>label</label><br>
 <input type="text" id="id">
 <input type="submit" 
        value="Submit Form">
</form>

// Script for checkFunction()

function checkFunction(){
 var targetField =
  document.getElementById("id");
 if(targetField.value.length === 0) {
  alert("an alert message");
  targetField.focus();
  targetField.style.background = "color";
  return false;
 }
 targetField.style.background = "color";
}

Notes to remember:

  • When the users press Submit Form, checkFunction() is activated.
  • This function checks to see whether the users have entered anything at all.
  • Inside the checkFunction(), the alert message can be anything you want it to be. It does not have to be an alert.
  • The statement - return false - prevents the form to be submitted since the form has not been filled out.
  • Notice that when the code is called in the onSubmit event after form, the return statement is also there. Without this statement, the form will be submitted even if the users have not entered the information.
  • The statement targetField.focus() is to make it user friendly by focusing on the form field the users have not entered.
  • After the targetField is focused, we set the background color to a different color to give further focus on the field we want the users to focus on.
  • After the return false statement, we change the style of the target field back to its original. This statement returns the field to the original color when the users are taken back to the form when they press submit again.
  • The syntax above uses document.getElementById, but we can use any other tagetting technique to get the element.

Example

Chapter 86 Concepts: Form Validation: Email.

This chapter shows us how to validate an Email.

Syntax 

// Check for Email Correction

function validateEmail(){
 var addressIsLegal = true;
 var eEntered = 
  document.getElementById("email").value;
  if(eEntered.indexOf(" ")!==-1){
   alert("Alert of spaces");
   return false;
  }
  if eEntered.indexOf("@")<1 || eEntered
     .indexOf("@")>eEntered.length-5) {
   addressIsLegal = false;
  }
  if(eEntered.indexOf(".")-eEntered
    .indexOf("@")<2 ||
    eEntered.indexOf(".") > eEntered
    .length-3){
     addressIsLegal = false;
  }
  if(addressIsLegal === false){
   alert("Ask for correction");
   return false;
  }
}

// Using regular Expression to check

function validateEmail(){
 var eEntered =
  document.getElementById("email").value;
  var emailCorrectPattern
   /^[\w\-\.\+]+\@[a-zA-Z0-9\.\-]+\
   .[a-zA-Z0-9]{2,4}$\;
 if(!eEntered.match(emailCorrectPattern)))
 { 
  alert("Ask for correction");
  return false;
 }
}

Note to remembers:

  • The syntax above test for a space which is an illegal character in an email. If there are other illegal characters in an email, we will have to use this same syntax to check for them.
  • The first if statement in the syntax checks to see if there is a space. If so, it is not correct.
  • The second if statement checks to see if there is a @ character in the email and there are fewer than 4 characters behind the @, and that the @ is not at the start of the email entered. If so, it is incorrect.
  • The last if statement checks to see if there is a dot behind the @ and that it is at least 2 to 4 characters following the dot. If not, it is incorrect.
  • The second function is to use regular expression to test for a correct email.

Chapter 87 Concepts: Exceptions: Try and Catch.

When a code is wirtten incorrectly and the function is run, nothing happens, but Javascript doesn't tell you why. You then have to read through the codes to try to figure it out. This chapter shows us how to use try and catch, so Javascript would report the errors in the codes.

Syntax 

function someFunction(){
 try{
  function code;
 }
 catch(error){
  alert(error);
 }
}

Notes to remember;

  • The function code is wrapped inside a try block
  • If there is an error, the catch statement will catch that error and alert what the error is. That is because the syntax above is written to dirett Javascript to report the error in an alert. We can choose to have any other actions wihtin the catch block.
  • Try and Catch are always a pair.
  • The try block is wrapped in curly brackets. The code within the try block is indented
  • The parameter "error" for the catch statement can be any name.
  • The Try and Catch pair has its limit. For example, if there is an error within the catch block, nothing will happen.
  • It is mostly used to catch underfined variables.

Demonstration and Example.

Chapter 88 Concepts: Exceptions: Throw.

The Try and Catch pair is mostly useful to catch underfined variables within the codes. That is usually a programmer's error.

This chapter shows us how to use Throw to catch an error mostly created by the users and throw the try and catch block a message that reports the users' errors.

Syntax 

function someCheckFunction(){
 try{
  codes to catch users' errors here;
  if(there is an error){
   throw "an error message";
  }
 }
 catch(error){
  alert(error);
 }
}

Notes to remember:

  • Codes to catch errors can be any codes like checking for correct emails, checking for zipcode digits, or checking for names covered in the previous chapters. It can be any checking function.
  • There is a Try and Catch pair here, but the catch block will report the message that the Throw statement sents it.
  • The real code checking for users' error is the code written by the programmers. It's not automatically checked by Javascript.

Chapter 89 Concepts: Handling Events Within Javascript.

Until now, we have been shown to use inline event handlings. For example we write a function as a value for an attribute like onClick for an HTML element. This chapter shows us to call a function within the Javascript code, not as an inline value for an HTML element.

Syntax 

// An HTML form input field

<input  type="button" 
        value="Click"
        id="ID">

// To handle event in Javascript

var input1 = 
 document.getElementById("ID");

// Handle various events

input1.onclick = someFunction;
input1.onsubmit = someFunction;

Notes to remember:

  • The syntax above only show the handling of two events, onclick and onsubmit, but we can use this technique to handle more events than just those two..
  • Notice that the events such as onclick, onsubmit are both in lowercase, not camelCase.
  • Notice the function someFunction does not have a pair of parentheses following it.
  • Whether we use inline handling or separate handling, the function codes are the same.
