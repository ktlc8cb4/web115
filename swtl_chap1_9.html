Chapter 1 Concept: Using the alert() Method.

We can use alert method to display a window box with a message. As the parameters for this method, we can pass in a string, a variable, a number, a math operation, or a mix of numbers, math operations, strings and variables.

Syntax

For strings: 

alert(place the message here inside 
  quotation marks);

For numbers: 

alert(place the number here);

For math expressions: 

alert(place math expressions here);

For a variable: 

alert(variable names);

For a mix of data types: 

alert(data type + data type);

Example

alert("Hello my Deer! We are eating 
  banana splits today!");

Chapter 4 Concept: Legal and Illegal Variable Names.

The key concept in this chapter talks about the standard of naming variables.

Concepts

  • Variable names cannot be a javascript keywords unless they are couple with some other words.
  • A variable name can only contain dollar sign, underscore, letters, and numbers. However, numbers cannot be the start of a variable name.
  • The standard for naming variables is using camelCase which means the first letter of the first word is lower case, and all the first letters of the subsequent words are capitalized.

Example and Demonstration

var abstract = 2;

Illegal: abstract is JS keyword.

var _$Hello1 = 3;

Legal: the undescore, dollar sign, letters, and numbers are all legal, and the variable doesn't start with a number.

var workHours = 3;

In Standard: workHours uses camelCase convention.

Chapter 6 Concept: Unfamiliar Math Operators.

Javascript uses the symbol ++ or -- next to a number to add or subtract one from that number. Placing these symbols before the number will result differently from placing it behind the number.

Syntax

With this syntax, only deerNum increases or decreases by 1.

var deerNum = a number;
var newDeerNum = deerNum++;
var newDeerNum = deerNum--;

With this syntax, both deerNum and newDeerNum increases or decreases by 1.

var deerNum = a number;
var newDeerNumDeer = ++newDeerNum
var newDeerNum = --newDeerNum

Example

var deerNum = 2 ;
var newDeerNum = deerNum++;
console.log(newDeerNum);

The answer is 2. Because the ++ is placed behind deerNum, newDeerNum does not benefit from the increment. Only deerNum benefits from it.

var deerNum = 2 ;
var newDeerNum = ++deerNum;
console.log(newDeerNum);

The answer is 3. Because the ++ is placed in front of deerNum, both newDeerNum and deerNum will increase to 3. The concept works the same way with -- .

Chapter 7 Concept: The Order of Math Expressions.

Javascript follows the the standard order of the math expressions - multiplication and division are done before addition and subtraction. To eliminate ambiguity, JS uses parentheses to define the order that an expression should follow.

Syntax

No quotation mark enclosure around any block to allow the flow of normal standard operation. 

var deerNum = a number + a number / 
  another number;

Enclose the block to be executed first inside a set of parentheses. 

var deerNum = (a number + a number) / 
  another number;

Example

var deerNum = 2 + 2 * 3;

The answer is 8. Multiplication is done before addition.

var deerNum = (2 + 2) * 3;

The answer is 12. Addition is done first because it is enclosed inside a pair of parentheses. Multiplication is the second operation.

Demonstration

Let's evaluate the expression below.

Chapter 8 Concept: Concatenating Strings.

Strings can be concatenated or added together to make a meaningful message. However, there are a few rules that need to be followed for the strings to display correctly.

Syntax 

var newDeer = "a string" + 
  "another string" + a variable;

- Strings can be concatenated by another string or a variable. Strings are enclosed by quotation marks whereas variables are not.

- Add spaces inside the quotation marks to give the string spaces where needed.

Demonstration

alert("Hello" + "my Deer!");

Notice there is no space after the word "Hello". The result will be - Hellomy Dear!

alert("Hello " + "my Deer!");

Notice there is a space after the word "Hello". The result will be - Hello my Dear!

alert("We have " + 2 + 2 + 
  " Vegan Deer here.");

Mixing numbers and strings will result in a string. Javascript will successfully make 2 + 2 = 22. The result will be - We have 22 Vegan Deer here.

Chapter 9 Concept: Using the prompt() method.

Like the alert() method, the promp() method will show a popup window displaying a message. The difference is the prompt() method asks for user input and saves the information to a variable. Inputs from the prompt() only result in strings even if a number is the input. This method allows for a default value to be stored in the designated variable in case the user doesn't give an input.

Syntax

The message displayed in the prompt box could be a string, a number, the content of a variable, or a misture of all those types.

var variableName = prompt("a string of 
  message.");
var variableName = prompt(a variable);
var variableName = prompt(a number + a 
  variable + "a string");

The prompt() method allows the option of using a default value if no input is received.

var variableName = prompt("a string of message", "default value");

Example

This variable newDeer will receive the value of whatever the user puts in as their name.

var newDeer = prompt("Please enter your 
  name, my Deer!");

Demonstration
