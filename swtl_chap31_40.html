Chapter 31 Concepts: Getting the Current Date and Time.

Using Date() to create a new Date object.

Syntax

Creating a new Date object.

var someDate = new Date();

A few points to remember:

  • Date objects are not strings, so you cannot use the methods for strings.
  • Date objects rely on the computer set Date and Time. If the Date and Time on the computer has not been set correctly, the Date object will also give an accurate value.

Example

Creating a Date object.

var deerDate = new Date();

Chapter 32 Concepts: Extracting Parts of the Date and Time.

The Date() object has several important methods to extract different pieces of information.

Syntax

Using one of these several methods to show the syntax. All of the methods follow the same syntax.

var someDate = new Date();
var someDay = someDate.getDay();

These are the highlighted methods.

  • getDay() - Get the day from the Date object.
  • getMonth() -Get the month from the Date object.
  • getDate() - Get the date from the Date object.
  • getFullYear() - Get the year from the Date object.
  • getHours() - Get the hour from the Date object. The hour is in military time.
  • getMinutes() - Get the minute from the Date object.
  • getSeconds() - Get the second from the Date object.
  • getMillisecond() - Get the millisecond from the Date object.
  • getTime() - Get milliseconds since midnight, January 1, 1970.

Example

The example shows only one method since all the other methods follow the same syntax.

var deerDate = new Date();
var deerHour = deerDate.getHours();

Chapter 36 Concepts: Passing Data to Functions.

A function can be set up to allow users to give it some data when it is called.

Syntax

function someFunction (parameter, 
  parameter, parameter) {
    Code to be executed 
}

Points to remember:

  • The parameters inside the parentheses are the placeholders for the users to pass data into.
  • The data passed into the function will follow the order of the parameters set in the function.
  • There is no limit to the number of parameters that can be set for a function.
  • The data passed into these parameters are called arguments.
  • The names you give the parameters automatically become variables for that functions.

Example

A example of a function that takes two parameters.

function deerMember(name, age){
  console.log(name + " " + age + ".");
}

//call the function and pass arguments
deerMember("Mark", 23);

    Chapter 39 and Chapter 40 Concepts: Switch Statement.

    Chapter 39 and 40 focus on swith statement. Switch statement work similar to the If Else statement. It can make the code easier if there are many conditions to meet.

    Syntax 

    // a variable is defined.

var someVar = value;

/* Use this known value to compare 
   in a switch statement. If someVar 
   equals the value for the case, 
   then the code for that case will 
   execute. */

switch(someVar) {
  case "value": 
    code to be execute;
    break;
  case "value: 
    code to be executed;
    break;
  case "value:
    code to be executed;
    break;
  default:
    code to be executed; 
}

    A few points to remember:

    • The variable someVar above is compared to the values corresponding to the case. If it matches the case value, the code for that case will be executed.
    • Use break; to stop the switch execution after the values match and the code executes successfully. If the break statement is not included, the codes for the remaining cases will all execute which is not something we want.
    • The last case of the switch statement is the default; which means if none of the cases match, the code for the default statement will execute. The default statement is optional. The default statement does not need a break; because it is usually the last execution of the switch statement; however, it is always safe to include break; for any last execution of the switch statement.

    Example 

    /* defining a variable to compare */

var deerName = "Mark";

/* compare deerName to all the cases */

switch(deerName) {
  case "Christy": 
    console.log("Hello " + deerName);
    break;
  case "Mark": 
    console.log("Hello " + deerName);
    break;
  case "Jennifer":
    console.log("Hello " + deerName);
    break;
  default:
    console.log("Well, hello there!");
}
